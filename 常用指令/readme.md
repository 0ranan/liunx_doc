### dmesg 命令

> 显示内核环形缓冲区的消息

```bash
# 1. 查看所有内核消息（可能很长）
dmesg

# 2. 分页查看（推荐！按空格翻页，q 退出）
sudo dmesg | less

# 3. 只看最后 10 行（快速定位最近事件）
sudo dmesg | tail -n 10
```


### wc 命令

> 如果不存在文件名，wc 命令就不会输出文件名

```bash
wc e1.sh
# 10  37 244 e1.sh
# 行数（lines） 单词数（words） 字符数（characters） 文件名称
```

```bash
wc -l e1.sh
# 10 e1.sh
# 行数（lines） 文件名称
```

### tar

> czvf；xzvf；压缩并归档；解压缩并还原
> **实际上在解压时 只需要输入 zvf 即可，tar会自动检测压缩类型**


```bash
tar -czvf e1.tar.gz e1.sh sshsrc # 压缩 e1.sh 文件 、sshsrc目录并归档为 e1.tar.gz
tar -xzvf e1.tar.gz # 解压缩 e1.tar.gz 文件并还原为 e1.sh 文件
```

#### 后缀含义

|后缀|含义|
|-|-|
| -z | 使用gzip压缩|
| -v | 显示详细信息|
| -f | 归档|
| -x | 解压缩|
| -j | 使用bzip2压缩|

### test 命令

```bash
test -f /etc/passwd && echo 'true' || echo 'false'
test -e e1.sh
# 检查 e1.sh 文件是否存在
test -f e1.sh
# 检查 e1.sh 文件是否是普通文件
test -d e1.sh
# 检查 e1.sh 文件是否是目录
test -r e1.sh
# 检查 e1.sh 文件是否可读
test -w e1.sh
# 检查 e1.sh 文件是否可写
test -x e1.sh
# 检查 e1.sh 文件是否可执行
```

|后缀|含义|
|-|-|
| -e | 文件存在|
| -f | 文件存在且是普通文件|
| -d | 文件存在且是目录|
| -s | 文件存在且大小大于0|
| -r | 文件存在且可读|
| -w | 文件存在且可写|
| -x | 文件存在且可执行|

## Linux核心概念

redhat 上 rpm相当于ubuntu的apt windows的winget

密码是放在 /etc/shadow 文件中的，而不是 /etc/passwd 文件中。

POSIX 协议是一种操作系统接口标准，定义了操作系统的行为和功能。它是由 IEEE 标准组织定义的，被广泛用于 Unix-like 系统（如 Linux、macOS 等）。

安装Linux时最少需要两个分区,分别是根分区(/)和交换分区(swap)。

常见 Linux 文件类型还有：目录(d)、字符设备(c)、块设备(b)、符号链接(l)、命名管道(p)、套接字(s)和链接文件(h)

Linux内核版本号（X.Y.Z）中包含的主要信息有 主要版本号（X）、次要版本号（Y）和维护版本号（Z）。

Linux下新建用户的uid默认从1000开始。

### umask

```text
Linux中文件权限掩码（umask）的默认值是: 0022
四位umask值分别表示：特殊权限位，用户权限、组权限、其他权限。
特殊权限位：
- 第1位：设置粘着位（sticky bit），用于目录，限制目录下文件的删除权限。
- 第2位：设置SUID位（setuid bit），用于可执行文件，使文件在执行时临时获得文件所有者的权限。
- 第3位：设置SGID位（setgid bit），用于目录，使目录下的文件继承目录的组权限。
```

### liunx 目录结构

| 目录位置| 含义|
|-|-|
| / | 根目录，所有文件和目录的起始点。 |
| /bin | 存放二进制可执行文件的目录，常用的系统命令都在这里。 |
| /boot | 存放系统引导程序的目录，包括内核文件和引导加载程序。 |
| /dev | 存放设备文件的目录，包括块设备和字符设备。 |
| /etc | 存放系统配置文件的目录，包括网络配置、服务配置等。 |
| /home | 存放用户主目录的目录，每个用户都有一个对应的子目录。 |
| /lib | 存放系统共享库文件的目录，包含了系统运行所需的动态链接库。 |
| /media | 存放可移动媒体设备（如 USB 驱动器）的挂载点目录。 |
| /mnt | 存放临时挂载文件系统的目录，常用于手动挂载文件系统。 |
| /opt | 存放可选应用程序的目录，第三方软件安装在这里。 |
| /proc | 虚拟文件系统，提供系统运行时的进程和系统信息。 |

## Shell基础与编辑器

| 运算符 | 说明 | 举例 |
| ------ | ------------------------------------------------- | ------------------------------------------------- |
| -eq | 检测两个数是否相等，相等返回 true。                |  [ $a -eq $b ] 返回 false。|
| -ne | 检测两个数是否不相等，不相等返回 true。                   |  [ $a -ne $b ] 返回 true。|
| -gt | 检测左边的数是否大于右边的，如果是，则返回 true。                   |  [ $a -gt $b ] 返回 false。|
| -lt | 检测左边的数是否小于右边的，如果是，则返回 true。                   |  [ $a -lt $b ] 返回 true。|
| -ge | 检测左边的数是否大于等于右边的，如果是，则返回 true。                  |  [ $a -ge $b ] 返回 false。|
| -le | 检测左边的数是否小于等于右边的，如果是，则返回 true。                  |  [ $a -le $b ] 返回 true。|

### 通配符

| 通配符                 | 含义说明                                              |
| ------------------- | ------------------------------------------------- |
| `*`                 | 匹配 **任意长度** 的字符（包括空字符），**不包括隐藏文件**（即以 `.` 开头的文件）。 |
| `?`                 | 匹配 **任意单个字符**。                                    |
| `[abc]`             | 匹配 **括号中任意一个字符**（a 或 b 或 c）。                      |
| `[a-z]`             | 匹配 **指定范围内任意一个字符**（a 到 z 之间的任意小写字母）。              |
| `[!abc]` 或 `[^abc]` | 匹配 **不在括号中的任意一个字符**（取反）。                          |
| `{str1,str2}`       | **大括号展开**，不是通配符，但常用于生成多个字符串组合（由 Bash 展开）。         |

cron 服务是一个在后台运行的守护进程，用于定时执行任务。它通常在系统启动时自动启动，并在系统关闭时自动停止。

 | 字段 | 取值范围            | 特殊符号                           |

| 日  | 1-31            | 同上                             |

### gcc
| 后缀                             | 含义 / 用途                | 处理方式                   |
| ------------------------------ | ---------------------- | ---------------------- |
| `.c`                           | C 源文件                  | 预处理 → 编译 → 汇编 → 链接     |
| `.i`                           | 已预处理的 C 源（纯 C）         | 直接编译 → 汇编 → 链接         |
| `.ii`                          | 已预处理的 C++ 源            | 同上，走 C++ 前端            |
| `.s`                           | 汇编源文件（人可读）             | 汇编 → 链接                |
| `.S`                           | 需预处理的汇编源（可含宏、#include） | 预处理 → 汇编 → 链接          |
| `.h` / `.hpp`                  | 头文件                    | 不被直接编译，由 `#include` 引入 |
| `.o`                           | 目标文件（已汇编，未链接）          | 参与最后链接                 |
| `.a`                           | 静态库（ar 打包的 .o 集合）      | 链接时整包复制进可执行文件          |
| `.so` / `.so.x`                | 共享库（动态库）               | 运行时动态加载/链接             |
| `.dylib`                       | macOS 动态库（与 .so 类似）    | 略                      |
| `.l`                           | lex 词法源文件              | 先 `flex` 生成 .c         |
| `.y`                           | yacc 语法源文件             | 先 `bison` 生成 .c        |
| `.cc` / `.cpp` / `.cxx` / `.C` | C++ 源文件                | 走 g++ 前端               |
| `.m`                           | Objective-C 源          | 走 gcc -objc 前端         |
| `.mm`                          | Objective-C++ 源        | 走 g++ -objc++ 前端       |
| `.S`                           | 需预处理的汇编                | 见上                     |

## C语言

### pipe()函数

单向数据通道：pipe() 在内核创建一对文件描述符（fd[0] 读，fd[1] 写）。

### fork()函数

创建子进程：fork() 调用会创建一个新进程，子进程是父进程的副本。

### signal函数

信号处理：signal() 用于注册信号处理函数，当接收到指定信号时，调用该函数。该函数需要传递两个参数：

- 信号编号（如 SIGINT、SIGTERM 等）
- 信号处理函数的指针（如 void handler(int signum)）

```text
常见中断信号：

- SIGINT（Ctrl+C）：终止当前进程
- SIGQUIT（Ctrl+\）：终止进程并生成 core dump 文件
- SIGTERM（默认）：终止进程
- SIGKILL（强制终止）：立即终止进程
- SIGHUP（终端断开）：通知进程终端已断开
- SIGSTOP（暂停进程）：暂停进程
- SIGCONT（继续进程）：继续暂停的进程
```

### pause函数

暂停进程：pause() 使进程进入暂停状态，直到接收到一个信号。
